# 17track Sign Generation

Found how 17track builds the `sign` field for their `/track/restapi` endpoint.

## The problem

17track's API requires a `sign` parameter that's generated by a WASM module running in the browser. Without a valid sign, the API always returns `code:-14`. You can't just fake it because it's a real browser fingerprint.

## What the sign actually is

It's generated by a Rust-compiled WASM binary (~125KB). The WASM `get_fingerprint()` function collects browser data like canvas fingerprint, WebGL renderer, navigator properties, screen resolution, timezone, hardware concurrency, etc. The output is ~1137 characters, always starts with `MjAyNTA3MjJUMTE6Mjg6NDVa` (base64 of the WASM build timestamp).

The sign is not tied to a specific tracking number. One sign works for any number within the same session.

## How I solved it

1. Load the 17track page in **Browserless.io** (cloud browser, stealth mode)
2. Hook `WebAssembly.instantiate` before page load to capture the WASM exports
3. Hook `window.fetch` to capture the `Last-Event-Id` header (hex-encoded fingerprint)
4. Page loads, WASM runs, API returns `code:-14` (CAPTCHA required)
5. Solve the image CAPTCHA using **Gemini 2.5 Flash** vision API
6. After CAPTCHA is solved, the session is unlocked
7. Now I can call `get_fingerprint()` directly and use the sign to query any tracking number

## The API call

```json
{
  "data": [{"num": "TRACKING_NUMBER", "fc": 0, "sc": 0}],
  "guid": "",
  "timeZoneOffset": -420,
  "sign": "MjAyNTA3MjJUMTE6Mjg6NDVa..."
}
```

Headers need `Last-Event-Id` from the original session and session cookies.

## CAPTCHA solving

17track shows an image grid CAPTCHA: one question image and 9 answer images. You need to pick which answers match the same type of animal/object as the question. I extract all images as base64 from the page canvas and send them to **Gemini 2.5 Flash** to identify the matching ones. Gemini runs from inside the browser context (called directly from Browserless) so there's no CSP issues.

## Performance

- First request: ~15-20s (page load + CAPTCHA + Gemini call)
- Every request after: ~0.5s (direct WASM API call, no browser needed)

## Services & tools used

- **Browserless.io** — cloud browser service (stealth mode). Used to load the 17track page and run WASM in a real browser environment. No local Puppeteer/Chrome needed
- **Gemini 2.5 Flash** (Google AI) — vision API for solving the image grid CAPTCHA. Called from inside the browser via Browserless
- No proxy needed for this one — Browserless handles the browser fingerprinting

## Note about chinapostaltracking.com

That site uses 17track's API under the hood. Same sign mechanism, same endpoints. So solving 17track's sign covers that too.
